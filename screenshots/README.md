# Screenshots Guide

Place the following screenshots in this directory for the README.

## Required Screenshots

1. **overview.png** - Main treemap view showing folder-level hierarchy
   - Capture: Full treemap of a medium-sized project
   - Resolution: 1920x1080 or higher
   - Show: Folder names, color gradient, visible structure

2. **navigation.gif** - Animated demo of clicking/drilling down
   - Capture: Click folder → see files → click file → see blocks
   - Duration: 5-10 seconds
   - Format: Optimized GIF or MP4
   - Show: Mouse cursor, breadcrumb updates

3. **colors.png** - Color gradient demonstration
   - Capture: Treemap with clear size differences
   - Highlight: Red (large) to blue (small) gradient

4. **click-to-code.gif** - Ctrl+Click navigation demo
   - Capture: Ctrl+Click on code block → editor opens at line
   - Duration: 3-5 seconds
   - Show: Mouse cursor, editor jump

5. **hierarchy.png** - Multi-level view showing folders/files/blocks
   - Capture: Treemap with visible folder, file, and block levels

6. **folder-view.png** - Top-level folder view
   - Capture: Fresh treemap showing project folders

7. **file-view.png** - Drilled into folder showing files
   - Capture: After clicking a folder

8. **block-view.png** - Deepest level showing code blocks
   - Capture: After clicking a file
   - Show: Functions/classes with line numbers

9. **dark-theme.png** - Dark theme compatibility
   - Capture: Treemap with VS Code dark theme

## Screenshot Guidelines

- **Format**: PNG for static, GIF/MP4 for animated
- **Size**: Keep under 2MB each (optimize with TinyPNG)
- **Content**: Use real code (not sensitive/proprietary)
- **Consistency**: Same project across screenshots when possible

## Capture Tools

- **Windows**: `Win+Shift+S` for screenshots
- **GIF recording**: ScreenToGif, LICEcap
- **Screen recording**: OBS Studio → convert to GIF
